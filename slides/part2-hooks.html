Qu'est-ce qu'un hook ?

Les hooks sont des commandes shell qui s'exécutent à des points précis du cycle de vie de Claude Code — toujours, quoi que décide le modèle.

Prompt / Instructions

Claude PEUT décider de le faire… ou non.

Hook

Claude VA le faire, systématiquement.

💡 Analogie : pensez aux hooks Git (pre-commit, post-push) — sauf qu'ici, c'est pour votre agent IA.

Les événements du cycle de vie

Début de session

SessionStart

Démarrage / reprise de session

non-bloquant

UserPromptSubmit

Avant que le prompt atteigne Claude

peut bloquer

Exécution d'outils

PreToolUse

Avant l'exécution d'un outil (Bash, Write…)

peut bloquer

PostToolUse

Après l'exécution d'un outil

feedback possible

Fin de session

Stop

Quand Claude a fini de répondre

peut forcer continuer

SubagentStop / PreCompact

Fin d'un subagent / avant compaction

selon l'événement

Handlers & exit codes

Les 4 types de handlers
command

Script shell — ~90% des cas, <5ms

http

Requête HTTP POST vers un endpoint

prompt

Évaluation par Claude Haiku

agent

Spawn d'un subagent avec outils — 30-60s

Le mécanisme d'exit code 
exit 0  # ✅ Autoriser, continuer
exit 1  # ⚠️  Erreur → feedback pour Claude
exit 2  # 🚫 BLOQUER l'action

Tout ce que vous écrivez sur stderr est transmis à Claude comme feedback.

Structure de configuration

Les hooks vivent dans les fichiers de settings JSON, à plusieurs niveaux :

Niveaux de configuration
USER ~/.claude/settings.json tous les projets PROJET .claude/settings.json versionnable en git LOCAL .claude/settings.local.json non commité
Structure de base 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{ "type": "command", "command": "..." }]
      }
    ]
  }
}

Le champ matcher est une expression régulière sur le nom de l'outil.

🛠 TP 1 — Formatage automatique (5 min)

💡 Prérequis : macOS / Linux natif — Windows avec Git Bash. Les scripts .sh s'exécutent dans le même shell.

Créez .claude/settings.json à la racine du projet 
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "FILE=$(echo \"$CLAUDE_TOOL_INPUT\" | python3 -c \"import sys,json; d=json.load(sys.stdin); print(d.get('file_path') or d.get('path',''))\"); case \"$FILE\" in *.js|*.ts|*.jsx|*.tsx) npx --yes prettier --write \"$FILE\" 2>/dev/null;; esac"
      }]
    }]
  }
}

Test : demandez à Claude de créer un fichier test.js mal formaté — il doit être nettoyé automatiquement.

🛠 TP 2 — Hook de sécurité (7 min)

💡 Prérequis : macOS / Linux natif — Windows avec Git Bash. Python3 requis (installé par défaut sur macOS/Linux).

~/.claude/hooks/security-check.sh 
#!/bin/bash
INPUT=$(cat)
CMD=$(echo "$INPUT" | python3 -c \
  "import sys,json; d=json.load(sys.stdin); \
   print(d.get('tool_input',{}).get('command',''))")
[ -z "$CMD" ] && exit 0

for p in "rm -rf /" "DROP TABLE" "chmod 777"; do
  if echo "$CMD" | grep -qi "$p"; then
    echo "🚫 Bloqué : '$p'" >&2; exit 2
  fi
done
exit 0
chmod +x ~/.claude/hooks/security-check.sh
~/.claude/settings.json 
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/hooks/security-check.sh"
      }]
    }]
  }
}

Test : demandez à Claude d'exécuter rm -rf /tmp/../

🛠 TP 3 & 4 — Notification & Contexte (8 min)

💡 Prérequis : macOS / Linux natif — Windows avec Git Bash. Windows nécessite aussi le module PowerShell BurntToast pour les notifications (Install-Module BurntToast).

Notification native (cross-platform)

Fonctionne même sans terminal ouvert :

#!/bin/bash
MSG="Claude a terminé !"
case "$(uname)" in
  Darwin) osascript -e "display notification \"$MSG\" with title \"Claude Code\"" ;;
  Linux)  notify-send "Claude Code" "$MSG" ;;
  *)      powershell.exe -Command "New-BurntToastNotification -Text 'Claude Code','$MSG'" ;;
esac

Sauvegarder dans ~/.claude/hooks/notify.sh puis chmod +x

Injection de contexte au démarrage

① ~/.claude/hooks/context.sh

#!/bin/bash
echo "Dossier : $(pwd)"
case "$(uname)" in
  Darwin|Linux) echo "Date : $(date '+%Y-%m-%d %H:%M')"; ls ;;
  *) powershell.exe -Command "Get-Date; Get-ChildItem | Select Name" ;;
esac

② ~/.claude/settings.json — puis chmod +x context.sh

{
  "hooks": {
    "SessionStart": [{
      "matcher": "",
      "hooks": [{ "type": "command", "command": "~/.claude/hooks/context.sh" }]
    }]
  }
}
