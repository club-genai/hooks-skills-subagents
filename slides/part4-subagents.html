Qu'est-ce qu'un Subagent ?

Un subagent est un agent Claude séparé que votre agent principal peut spawner pour déléguer une tâche. Il a son propre contexte, ses propres permissions et son propre modèle.

Skill

Instructions chargées dans le contexte de l'agent courant — même processus, même modèle

Subagent

Processus entièrement séparé — contexte isolé, modèle et permissions propres

📋

Contexte propre

Window séparée — il explore 50 fichiers, le parent reçoit un résumé propre

⚡

Parallélisation

3 subagents travaillent en simultané

🛡️

Permissions différenciées

Lecture seule pour explorer, écriture pour implémenter

Optimisation coûts

Quand NE PAS utiliser les subagents

⚠️ Un subagent crée de l'overhead. Ne pas l'utiliser si le bénéfice ne compense pas le coût.

Coût & Performance

❌ Tâches courtes et simples

L'overhead du spawn dépasse le bénéfice de l'isolation.

⚠️ Trop de subagents définis

Dégrade la qualité du routing automatique.

Contraintes techniques

❌ Modifications sur les mêmes fichiers

Risque de conflits en écriture concurrente.

❌ Communication inter-agents

Les subagents ne peuvent pas se parler directement. Utilisez Agent Teams.

🛠 TP 1 — Fichier de test src/api/users.js

💡 Créer ce fichier puis lancer : @tech-writer, génère la doc pour src/api/users.js

const db = require('./db');

// GET /users
async function listUsers(filters = {}) {
  let q = 'SELECT * FROM users';
  if (filters.role) q += ` WHERE role = '${filters.role}'`;
  if (filters.limit) q += ` LIMIT ${filters.limit}`;
  return db.query(q);
}

// POST /users
async function createUser(data) {
  const { name, email, role = 'user' } = data;
  if (!name || !email) throw new Error('Missing fields');
  return db.query(
    'INSERT INTO users (name, email, role) VALUES (?, ?, ?)',
    [name, email, role]
  );
}

// PUT /users/:id
async function updateUser(id, data) {
  const fields = Object.keys(data).map(k => `${k} = ?`).join(', ');
  return db.query(`UPDATE users SET ${fields} WHERE id = ?`,
    [...Object.values(data), id]);
}

module.exports = { listUsers, createUser, updateUser };
📄 L'agent doit produire
• JSDoc sur chaque fonction • Paramètres + types • Exemples d'utilisation • README structuré
🔍 Bonus à repérer
• Injection SQL dans listUsers • Pas de validation dans updateUser
