Partie 1 : Les Hooks

15 MINUTES

Théorie

Que sont les Hooks en GenAI ?

Concept : Ce sont des points d'interception dans le cycle de vie d'une requête LLM (avant l'envoi du prompt, après la réception, en cas d'erreur).

⚙️ Pre-processing

  • Injection de contexte
  • Filtrage PII (données sensibles)
  • Formatage

📊 Post-processing

  • Validation JSON & parsing strict
  • Logging
  • Calcul de tokens/coûts
Architecture : Middleware pattern appliqué aux LLMs.
Workshop

Créer son premier Hook

Consigne : Implémenter un système de logging et de calcul de coût (token counting).

💻 L'exercice :

  • Créer une fonction d'appel LLM standard (ex: API OpenAI ou Gemini).
  • Ajouter un décorateur ou middleware (before_request / after_request).
  • Afficher dans la console le temps d'exécution et le coût estimé.

Takeaway : Les participants ont maintenant un framework de base pour monitorer toutes les actions de leur futur Agent.

Partie 2 : Les Skills

15 MINUTES

Théorie

Donner des "mains" au LLM

Concept : Un LLM seul est isolé (cut-off knowledge). Les Skills (ou Function Calling / Tools) lui permettent d'interagir avec le monde extérieur.

🔁 Fonctionnement :

  1. L'utilisateur pose une question.
  2. Le LLM répond avec un JSON demandant l'exécution d'une fonction précise.
  3. Le code local exécute la fonction.
  4. Le résultat est renvoyé au LLM pour formuler la réponse finale.
Workshop

Coder un Skill

Météo ou Base de données

Consigne : Transformer une fonction Python/JS classique en "Skill" compréhensible par le modèle.

💻 L'exercice :

  • Créer une fonction get_database_schema(table) ou get_weather(city).
  • Créer le schéma JSON (OpenAPI/JSON Schema) qui décrit cette fonction au LLM.
  • Faire une boucle simple : Prompt ➔ LLM ➔ Détection du Tool ➔ Exécution locale ➔ Réponse.

Partie 3 : Les Agents

20 MINUTES

Théorie

De la requête simple à l'autonomie

Un Agent = Cerveau + Mémoire + Skills + Planification

🔄 Pattern ReAct

L'agent réfléchit à voix haute (Thought), décide d'une action (Action), observe le résultat (Observation), et boucle jusqu'à la solution finale.

🎯 System Prompt

C'est l'ADN de l'agent : on y définit le persona, les règles strictes d'engagement et le format de sortie attendu.

Workshop

Construire la boucle ReAct

Consigne : Créer une boucle while autonome.

💻 L'exercice :

  • Fournir 2 ou 3 skills (ex: calculate, search_wikipedia).
  • Donner un problème nécessitant ces outils :
    "Quelle est la racine carrée de la population de Paris ?"
  • Coder la boucle d'orchestration qui parse les pensées et exécute les actions jusqu'à trouver Final Answer.

Partie 4 : Les Sub-Agents

20 MINUTES

Théorie

Orchestration et Multi-Agents

❌ Le problème

Un Agent Unique avec trop de skills = perte de focus, hallucinations fréquentes, prompts trop longs.

✅ La solution

La spécialisation. Un agent routeur (Manager) qui délègue à des agents spécialisés (Workers).

🕸️ Patterns de communication :

  • 👑 Hiérarchique
  • ➡️ Séquentiel
  • 🗣️ Débat
Workshop

L'équipe d'Agents

Consigne : Créer un Agent "Manager" qui délègue.

Exemple de workflow :

  1. Créer l'Agent A (Chercheur web) et l'Agent B (Rédacteur).
  2. Le Manager reçoit : "Écris un post LinkedIn sur le dernier modèle de Google".
  3. Le Manager appelle le Chercheur ➔ il renvoie les faits.
  4. Le Manager passe les faits au Rédacteur ➔ il produit le post.
  5. Le code affiche la trace de qui a fait quoi.

Conclusion & Ouverture

10 MINUTES

Théorie

Les limites des Agents actuels

⏳
Temps de latence

Les boucles ReAct sont lentes par nature.

💸
Coût des tokens

La facture explose avec la mémoire contextuelle répétée à chaque boucle.

🐛
Fragilité

Gestion délicate des erreurs de parsing JSON (outputs non conformes).

Le "Compound AI Engineering"

L'avenir n'est pas "Un modèle gigantesque qui fait tout", mais des "Systèmes Composés".

C'est l'assemblage de :

  • 🧩 Petits modèles spécialisés
  • ⚙️ Code déterministe (Hooks)
  • 📚 Bases vectorielles (RAG)
  • ⚡ Caches sémantiques
  • 🛡️ Validateurs de sortie (Guardrails)
